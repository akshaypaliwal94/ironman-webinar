  UI/UX PRO-MAX CHECKLIST
  Universal quality standard for all web projects
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Use this checklist before every deployment. Each item should
be verified on: 320px, 375px, 480px, 640px, 768px, 1024px,
1280px, 1440px viewports.

────────────────────────────────────────────────────────
 1. TYPOGRAPHY
────────────────────────────────────────────────────────
□  All headlines use clamp() — never fixed px at large sizes
   Format: clamp(MIN, FLUID, MAX)
   e.g. clamp(32px, 6vw, 80px)

□  Body copy minimum 14px on mobile, 15–16px on desktop

□  Line-height: body 1.65–1.8, headlines 1.0–1.15

□  No text smaller than 11px anywhere (legal/disclaimer: 12px min)

□  Font weights: 400 body, 600 semi-bold, 700–800 CTAs/headlines

□  Letter-spacing: tight on Bebas Neue (0.02–0.05em),
   wider on uppercase labels (0.08–0.12em)

□  No orphaned single words on last line of headings at any breakpoint

□  Text contrast ratio: 4.5:1 minimum (WCAG AA)
   — White on navy: ✓
   — rgba(255,255,255,0.4) on navy: borderline — use sparingly

────────────────────────────────────────────────────────
 2. RESPONSIVE BREAKPOINTS (mobile-first mindset)
────────────────────────────────────────────────────────
□  360px — minimum support (no horizontal scroll)
□  480px — small mobile (most Android phones)
□  640px — large mobile / iPad mini portrait
□  768px — iPad portrait / large tablet
□  860px — tablet landscape / small laptop
□  1024px — laptop
□  1280px — standard desktop
□  1440px — large desktop (max-width containers)

□  Every multi-column grid collapses gracefully at the right
   breakpoint — never force 2 columns below 640px unless
   explicitly tested

□  No horizontal overflow at any viewport — test with
   overflow-x: hidden on body as a sanity check

□  Padding/margins scale down on mobile:
   Desktop: 80–100px vertical, 24px horizontal
   Mobile:  40–56px vertical, 16px horizontal

────────────────────────────────────────────────────────
 3. TOUCH & INTERACTIVITY
────────────────────────────────────────────────────────
□  All tap targets minimum 44×44px (Apple HIG / Material)
   — Buttons: padding must create ≥44px height
   — Icon-only buttons: explicit width/height ≥40px
   — Text links: add padding-top/bottom: 8px minimum
   — Form inputs: height ≥44px

□  Hover states defined for all interactive elements

□  No hover-only interactions on touch devices
   (animations/reveals must also work on tap)

□  :active state on all buttons (scale down 0.95–0.97)

□  Focus styles visible for keyboard navigation
   (outline or box-shadow on :focus-visible)

□  No double-tap zoom issues (use touch-action: manipulation
   on buttons/links if needed)

────────────────────────────────────────────────────────
 4. ANIMATION & MOTION
────────────────────────────────────────────────────────
□  Idle animations (breathe, float, shimmer) on CTAs and
   key elements to draw attention

□  Spring easing on interactive elements:
   cubic-bezier(0.34, 1.56, 0.64, 1) — satisfying overshoot

□  Hover: lift + scale + glow (translateY + scale + box-shadow)

□  Active/click: press down scale(0.95–0.97), duration 80–100ms

□  Staggered entrance animations (delay 0.1–0.2s per item)

□  Scroll-driven animations where appropriate
   (IntersectionObserver, reveal class pattern)

□  prefers-reduced-motion media query applied to ALL
   idle/entrance animations:
   @media (prefers-reduced-motion: reduce) { animation: none }

□  No animations longer than 0.6s for UI interactions
   (scroll effects can be slower)

□  Shimmer sweep on CTAs: diagonal gradient, periodic play

□  Arrow nudge on CTA buttons: elastic bounce hint

────────────────────────────────────────────────────────
 5. CTA BUTTONS
────────────────────────────────────────────────────────
□  Primary CTA: full-width on mobile (max-width on desktop)

□  Button height: ≥52px on desktop, ≥48px on mobile

□  Font: bold (700–800), never less than 14px

□  At least 3 animation states: idle breathe / hover lift / active press

□  Shimmer sweep on idle (plays every 3–5s)

□  Arrow/icon animates on hover (translate or rotate)

□  Hover: translateY(-4 to -6px) + scale(1.02–1.03) + deeper shadow

□  Box-shadow: multi-layer (near + far) matching brand color

□  On mobile: reduce padding (14–15px vertical, 20–24px horizontal)

□  Never href="#" in production — always a real destination

────────────────────────────────────────────────────────
 6. ICONS & VISUAL ELEMENTS
────────────────────────────────────────────────────────
□  Icon containers: gradient background (not flat color)
   Linear-gradient at 135–140deg for depth

□  Inner gloss: ::after pseudo with top-half white gradient
   rgba(255,255,255,0.2) → transparent

□  Idle float animation on decorative icons (3–4s, 5px travel)

□  Stagger float delays per icon (0.5s apart)

□  Active/scroll-triggered icons: spring scale + multi-ring glow

□  Icon sizes use clamp() or are tested at all breakpoints

□  SVG icons: strokeWidth="2–2.5", round caps/joins

□  Emoji icons in containers: container scales, not raw emoji

□  Touch: icon container ≥40×40px (ideally 44×44px)

────────────────────────────────────────────────────────
 7. LAYOUT & SPACING
────────────────────────────────────────────────────────
□  Max-width containers: 640px (copy), 780px (FAQ/centered),
   900px (standard), 980px (hero), 1200px+ (full sections)

□  Section vertical padding scales:
   Desktop: 80–100px top/bottom
   Tablet:  60–72px
   Mobile:  40–56px

□  Grid gaps scale: 24px desktop → 16px tablet → 12px mobile

□  Cards: border-radius 16–20px, consistent shadow system

□  Consistent card shadow: var(--card-shadow)
   = 0 2px 12px rgba(0,0,0,0.07)

□  Hover on cards: translateY(-2 to -4px) + deeper shadow

□  Section dividers: gradient lines or border-top, not hard lines

────────────────────────────────────────────────────────
 8. COLOR & VISUAL HIERARCHY
────────────────────────────────────────────────────────
□  Primary accent used sparingly — CTAs, key icons, underlines

□  Background alternates: navy → white/off-white → navy
   (creates breathing rhythm through the page)

□  Red accent top border on each section (4px gradient line)

□  Text opacity layers for hierarchy:
   Primary: #fff or var(--text-dark)
   Secondary: rgba(255,255,255,0.75) or var(--text-mid)
   Tertiary: rgba(255,255,255,0.4) or var(--text-light)
   Disabled/legal: rgba(255,255,255,0.28–0.35)

□  Pill badges: brand color bg + white text, border-radius 100px

□  No pure black (#000) — use var(--navy) or var(--text-dark)

────────────────────────────────────────────────────────
 9. SCROLL & PAGE EXPERIENCE
────────────────────────────────────────────────────────
□  Scroll reveal (IntersectionObserver) on all major sections

□  Threshold: 0.1 (10%) so elements reveal early enough

□  Reveal: opacity 0→1 + translateY(26px→0), 0.55s ease

□  Stagger reveals with reveal-delay-1/2/3/4 classes

□  DO NOT put reveal class on items that may already be in
   viewport on page load (first 2 items of a list etc.)

□  Scroll-driven line/progress animations use requestAnimationFrame
   or passive scroll listeners

□  html { scroll-behavior: smooth } for anchor links

□  No layout shifts (CLS) from lazy-loaded images or fonts

────────────────────────────────────────────────────────
 10. IMAGES & MEDIA
────────────────────────────────────────────────────────
□  All images use Next.js <Image> with fill or explicit dimensions

□  Portrait images: objectPosition "center top" to show faces

□  Landscape images in portrait containers: objectFit "contain"
   (shows full image, dark bg fills gaps invisibly)

□  Alt text on all images (descriptive, not decorative empty)

□  Priority prop on hero/LCP images

□  Image containers have explicit aspect-ratio to prevent CLS

□  WebP/AVIF used where possible (Next.js handles automatically)

────────────────────────────────────────────────────────
 11. FORMS & INPUTS
────────────────────────────────────────────────────────
□  Input height ≥44px

□  Font-size on inputs ≥16px (prevents iOS zoom on focus)

□  Clear focus rings (box-shadow or outline)

□  Labels visible (not just placeholder text)

□  Error states styled (red border + message)

□  Submit button: full CTA treatment (not default browser button)

────────────────────────────────────────────────────────
 12. MODALS & OVERLAYS
────────────────────────────────────────────────────────
□  Overlay: rgba(dark, 0.8–0.9) + backdrop-filter: blur(4–8px)

□  Modal card: border-radius 16–20px, padding 32–40px desktop,
   24px mobile

□  Animate in: fade + scale(0.92→1), 0.3s ease

□  Animate out: reverse (scale back, fade), then unmount

□  On mobile: reduce padding, check content fits viewport height

□  sessionStorage/localStorage gate to prevent re-showing

□  Accessible: trap focus inside modal, ESC to close (if allowed)

□  Force-choice modals: no close button, clear visual hierarchy

────────────────────────────────────────────────────────
 13. FOOTER
────────────────────────────────────────────────────────
□  Dark background (matches hero) for visual bookending

□  Brand name + tagline on left, social icons on right

□  Social icon buttons: 40×40px circles, hover glow effect

□  Disclaimer block: legal copy, muted opacity, 12px minimum

□  Copyright + links row at bottom

□  Mobile: stack to single column, center-aligned

□  Links have enough padding for touch (padding: 4px 0 minimum)

────────────────────────────────────────────────────────
 14. PERFORMANCE
────────────────────────────────────────────────────────
□  No blocking animations on initial render

□  CSS animations use transform/opacity only (GPU composited)

□  No layout-triggering props in animations
   (avoid animating: width, height, top, left, padding, margin)

□  IntersectionObserver: disconnect after observing all elements

□  Scroll listeners: { passive: true }

□  Heavy components lazy-loaded where possible

□  Fonts: display: swap to prevent FOIT

────────────────────────────────────────────────────────
 15. ACCESSIBILITY
────────────────────────────────────────────────────────
□  aria-label on icon-only buttons

□  aria-expanded on accordion toggles

□  aria-hidden="true" on decorative SVGs

□  Semantic HTML: <section>, <nav>, <main>, <footer>, <button>

□  Never use <div> for clickable elements — use <button> or <a>

□  prefers-reduced-motion respected for all animations

□  Color alone never conveys information (icons + color)

□  Skip-to-content link for keyboard users (optional but ideal)

────────────────────────────────────────────────────────
 16. FINAL CHECKS BEFORE DEPLOY
────────────────────────────────────────────────────────
□  All href="#" replaced with real URLs or onClick handlers

□  No console.log statements in production code

□  No placeholder images (broken img tags)

□  All links tested (internal routes + external)

□  sessionStorage/localStorage keys don't conflict across funnels

□  Correct meta title and description in layout.tsx

□  OpenGraph image set

□  favicon set

□  Build passes without TypeScript errors

□  Test on real iOS Safari and Android Chrome (not just desktop DevTools)

□  Test landscape orientation on mobile

□  Test with system font size increased (accessibility setting)
